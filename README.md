# Blenny-ts

<p align="center">
  <img src="BlennyMascot-TS.svg" alt="Blenny Mascot" width="250">
</p>

Build real-time, hypermedia-driven web apps in a single binary — with
zero-config module discovery and automatic background task supervision.

A Deno/TypeScript port of [Blenny](https://github.com/NathanFrund/blenny), a
hypermedia-driven, real-time, single-binary web platform.

## Features

- **Zero-config module discovery** — drop a `.ts` file into `src/modules/` and
  it loads automatically. Route registration, event subscriptions, and lifecycle
  hooks are all self-contained.
- **Real-time push** — SSE connections with DataStar signal merging,
  intent-filtered delivery, and automatic reconnection during deploys.
- **Single binary** — compile everything (framework + modules + static assets)
  into one executable with `deno task compile`.
- **Hypermedia-driven** — JSX rendering with a Conduit that wraps full-page
  loads in a layout and returns fragments for HTMX/DataStar swaps.
- **Pluggable auth** — swap auth strategies by renaming directories. Ships with
  a reference in-memory auth module (SHA-256, JWT cookies, role-based access).
- **Supervised background tasks** — TaskSupervisor manages recurring work with
  automatic exponential backoff, jittered retries, per-task error hooks, and
  lifecycle-aware cleanup.
- **OpenTelemetry** — built-in tracing and metrics with configurable export.

## Quick Start

```bash
git clone <repo> && cd blenny-ts
deno task dev
```

Opens on `http://localhost:3000` by default.

### Configure

Supply overrides via any priority order:

```bash
# 1. CLI args
deno run --allow-net --allow-env --allow-read main.ts --server.port=8080

# 2. Environment variables
BLENNY_SERVER_PORT=8080 deno task dev

# 3. blenny.json (in working directory)
echo '{ "server.port": "8080" }' > blenny.json

# See blenny.example.json for all keys
```

## Structure

```
src/
  core/        Framework internals (hub, conduit, auth, config, database)
  modules/     Drop-in modules auto-discovered at startup
main.ts        Server entrypoint
tests/         Test suite matching src/ layout
```

Toggle which modules are active by renaming directories in `src/modules/` — add
or remove a `.` prefix to enable or disable modules without deleting code.

## Key Concepts

| Concept            | Description                                                                                                  |
| ------------------ | ------------------------------------------------------------------------------------------------------------ |
| **Modules**        | Self-contained route + subscription units. Drop a `.ts` file into `src/modules/` and it loads automatically. |
| **TransportHub**   | Manages SSE and WebSocket connections. Broadcasts messages to clients with intent-based filtering.           |
| **Conduit**        | HTMX/DataStar-aware JSX renderer. Wraps content in a layout on full page loads, returns fragments on swaps.  |
| **Config**         | Composite provider — CLI > env > `blenny.json` > embedded defaults. All dotted keys.                         |
| **Auth**           | Pluggable via modules. Ships with a reference in-memory auth module (SHA-256, registration, JWT cookies).    |
| **TaskSupervisor** | Named recurring tasks with exponential backoff, per-task error hooks, and runtime introspection.             |

## Build

```bash
deno task compile
./blenny-ts --server.port=8080
```

Compilation generates `src/modules/manifest.ts` — a static import manifest that
bundles all active (non-dot-prefixed) modules into the binary. The manifest is
auto-generated by `tools/generate-manifest.ts` and runs as the first step of
`deno task compile`.
